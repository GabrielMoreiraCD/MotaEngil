1. Lacunas de Dados (Esperando a mota enviar) 
Os clusters atuais (petrobras_rag_teoria e normas_tecnicas_publicas) fornecem a base qualitativa (o "como fazer" e "qual padrão seguir"). Para gerar quantitativos (materiais e pessoal), o sistema precisa de três novos conjuntos de dados:

Documentos de Projeto (Inputs Dinâmicos): O sistema precisa ser capaz de receber e processar os documentos citados no escopo no momento da inferência. Desenhos de Engenharia (DE), Isométricos (IS) e Memoriais Descritivos (MD) contêm as métricas exatas. Isométricos de tubulação possuem uma "Lista de Materiais" (LM) nativa no próprio desenho que precisa ser lida.

Tabelas de Produtividade (Homem-Hora / HH): O modelo de linguagem não possui precisão intrínseca para estimar tempo de trabalho. É necessário ingerir tabelas de composição de preços e produtividade (ex: Tabelas TCPO ou métricas internas da Petrobras de HH por polegada de solda, HH por metro quadrado de pintura, HH por quilograma de suporte montado).

Catálogo Padrão de Materiais: Um banco de dados relacional (SQL ou planilha) com a codificação de materiais da Petrobras. O RAG precisará cruzar a descrição "Linha 3/4 polegada" com o código de estoque correto para aquele tipo de fluido/pressão (especificação de tubulação).

2. Arquitetura do Agente Delineador
A arquitetura deve evoluir do script linear rag_engine.py para um grafo de estados (utilizando LangGraph). 
O processamento deve ocorrer em múltiplas etapas autônomas:

A. Sub-Agente de Triagem (Parser de Escopo)

Função: Ler o documento principal (ex: "DE - LC-029") e extrair a lista de tarefas ("interligação", "fabricar suportes", "pintura") e as chaves dos documentos referenciados (ex: MD-3010.68-1200-941-M9C-003, IS-3010.68-1200-200-M9C-019).

B. Sub-Agente de Visão Computacional (Extração de Isométricos)

Função: Utilizar a infraestrutura já existente no módulo vision_engine.py (modelo Qwen-VL) para ler os arquivos IS e DE referenciados.

Ação Crítica: O prompt do modelo de visão deve ser estritamente focado em localizar a tabela "Lista de Materiais" no canto do desenho isométrico e retornar um JSON estruturado com: Item, Quantidade, Diâmetro, Descrição (ex: Tubo, Flange, Válvula, Curva).

C. Sub-Agente de Validação Normativa (RAG Atual)

Função: Consumir as tarefas listadas e buscar as normas aplicáveis no cluster normas_tecnicas_publicas e normas_tecnicas_privadas.

Exemplo: Para a tarefa "Realizar preservação e pintura das conexões", o agente utiliza o roteador para buscar a norma de pintura (ex: N-1735 ou N-13)
aplicável a conexões offshore, definindo o tipo de tinta, espessura e esquema, adicionando esses insumos à lista de materiais.

D. Sub-Agente de Estimativa (Cálculo de HH e Pessoal)

Função: Cruzar as quantidades extraídas (polegadas de diâmetro para soldar, área para pintar, peso de suportes) com as tabelas de produtividade.

Exemplo lógico: Se o isométrico aponta 10 soldas em tubo de 3/4", o agente calcula 10 x (produtividade HH/solda 3/4") para definir o número de
soldadores e encanadores necessários.

3. Necessidades Técnicas e Implementação
Para implementar este fluxo, as seguintes alterações técnicas são necessárias no código atual:

Implementação de Tool Calling (Function Calling): 
O LLM roteador não deve apenas decidir filtros vetoriais, mas sim invocar ferramentas Python específicas. As ferramentas necessárias incluem:

extrair_tabela_materiais(documento_id): Aciona o VLM no PDF do isométrico.

consultar_norma(termo_tecnico): Executa a busca no Qdrant.

calcular_homem_hora(tipo_servico, quantidade): Consulta o banco de dados de produtividade.

Memória de Estado (State Management): Durante a execução do pipeline, o sistema deve manter um objeto de estado (ex: um dicionário Python) que acumula progressivamente a lista_de_materiais e a lista_de_pessoal.

Processamento de Notas e Pendências: O agente de triagem deve possuir lógica para avaliar as restrições (ex: "Painel e amostrador 1 ausentes", "Parte do projeto já executado"). Isso obriga o sistema a aplicar filtros de exclusão sobre os materiais que o isométrico original pediria, calculando os recursos apenas para o escopo pendente.

4. Fluxo de Execução Recomendado (Pipeline de Inferência)
Input: O usuário submete o arquivo DE - LC-029 e os anexos (IS, MD).

Parsing: O LLM lê o DE e identifica: Tarefas = [Interligação, Fabricação de Spool, Pintura]. 
Documentos associados = [IS-019, IS-026]. 
Pendências = [Considerar apenas spool F-B21-507].

Extração de Desenhos: O VLM processa os isométricos IS-019 e IS-026, retornando flanges, tubos e válvulas de 3/4".

Consulta Regulatória: O RAG é consultado para definir materiais consumíveis extras
(eletrodos específicos para o material do tubo, esquema de pintura para conexões montadas no campo).

Cálculo: O sistema aplica os índices de produtividade sobre as quantidades e define que serão necessários 1 Soldador TIG/SMAW,
1 Encanador Industrial e 1 Pintor por X dias.

Output Final: Geração de um relatório em formato JSON ou tabela demarcando materiais permanentes, materiais consumíveis, equipamentos (máquina de solda) e efetivo (mão de obra).
